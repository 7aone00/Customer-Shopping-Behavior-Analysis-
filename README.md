End-to-End Customer Shopping Behavior Analysis
Python | SQL | Power BI | Data Analytics Portfolio Project
📌 Project Overview
This repository contains a full-cycle data analytics project designed to simulate a real-world business intelligence workflow. The goal is to transform raw retail data into actionable insights to help stakeholders understand customer segments, optimize marketing strategies, and drive revenue growth.

🛠️ Technical Workflow
The project was executed through four main stages:

Data Engineering & EDA (Python): * Cleaned and pre-processed 3,900 transaction records.

Handled missing values and performed exploratory data analysis (EDA).

Integrated Python with a SQL database using SQLAlchemy.

Advanced Analytical Querying (SQL): * Segmented customers into New, Returning, and Loyal tiers.

Executed complex queries to analyze revenue drivers and shipping impact.

Data Visualization (Power BI): * Built an interactive dashboard to track KPIs (Revenue, Avg Spend, Review Ratings).

Created dynamic filters to allow stakeholders to drill down by age, gender, and subscription status.

Strategic Reporting: * Developed a professional presentation and report summarizing key business recommendations.

📊 Key Insights Captured
Loyalty Impact: Subscribed customers contribute to 45% of total revenue and show a significantly higher average spend.

Top Performers: Clothing items (Blouses and Dresses) maintain the highest customer satisfaction scores.

Demographic Focus: The "Young Adult" segment is the primary revenue driver, contributing over $62k.

📂 Project Structure
Customer_Shopping_Behavior_Analysis.ipynb: Python notebook for cleaning and database connection.

customer_behavior_sql_queries.sql: SQL scripts for business-oriented analysis.

customer_behavior_dashboard.pbix: The interactive Power BI dashboard.

Business Problem Document.pdf: Project scope and requirements.

Customer-Shopping-Behavior-Analysis.pptx: Final presentation of insights.

🚀 How to Run the Project
Python: Run the .ipynb file to clean the data and load it into your SQL Server/PostgreSQL.

SQL: Execute the .sql queries to generate analytical views.

Power BI: Open the .pbix file (connect it to your SQL database if needed) to view the dashboard.
